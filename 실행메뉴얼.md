# 엑셀 거래처/고객 매칭 프로그램 실행메뉴얼

## 1. 프로그램 개요

본 프로그램은 **엑셀 A**와 **엑셀 B**의 거래처/고객 정보를 비교하여, 같은 거래처인지 **유사도 점수**로 판단하고 **고객명 일치 여부**를 별도로 확인한 뒤 **엑셀 C** 결과 파일을 생성합니다.

---

## 2. 프로그램 목적

- A 파일의 거래처가 B 파일 중 어느 거래처와 가장 유사한지 자동 매핑
- 거래처 매칭 결과와 고객명 일치 결과를 분리하여 제공
- 검수가 필요한 건(`검수필요`)과 근접 후보를 함께 제공하여 수작업 검증 시간 단축

---

## 3. 주요 기능

- 거래처명/시도/시군구/주소/전화번호 기반 유사도 점수 계산
- 거래처코드·고객코드·진료과목명은 C 파일에 A/B 원본값으로 함께 출력 (유사도 미사용)
- 고객명 100% 일치 여부 별도 판정 (진료과목명은 동명이인 구분 참고용)
- 고객명 일치 시 거래처 점수 +3 가점 (기본 75점 이상일 때)
- 시도명 우선 후보 검색
- 근접 후보 최대 3건 출력 (대표 후보와 5점 이내)
- 실행 로그 자동 저장 (`logs` 폴더)
- 콘솔 창에 진행률/오류 표시

## 5. 폴더 구조 설명

프로그램을 사용할 때 아래 구조를 유지해야 합니다.

```text
excel_matcher/
 ├─ excel_matcher.exe    ← 실행 파일
 ├─ input/               ← 입력 파일 폴더
 ├─ output/              ← 결과 파일 생성 폴더
 ├─ logs/                ← 실행 로그 폴더
 └─ rules/               ← 매칭 규칙 JSON 폴더
     ├─ sido_alias.json
     └─ company_remove_words.json
```

> `excel_matcher.exe`와 `rules` 폴더는 **반드시 같은 위치**에 있어야 합니다.

---

## 6. 입력 파일 준비 방법

### 6-1. 파일 위치

`input` 폴더에 아래 2개 파일을 넣습니다.

- `input/A.xlsx`
- `input/B.xlsx`

### 6-2. 파일명 규칙

파일명은 **반드시** 아래 이름을 사용해야 합니다.

| 파일명 | 설명 |
|--------|------|
| `A.xlsx` | 기준(원본) 데이터 |
| `B.xlsx` | 비교 대상 데이터 |

### 6-3. 필수 컬럼

A, B 파일 모두 아래 9개 컬럼이 **같은 이름**으로 있어야 합니다.

| 순서 | 컬럼명 | 유사도 사용 | 설명 |
|------|--------|-------------|------|
| 1 | 거래처코드 | X | 거래처 고유 키 |
| 2 | 거래처명 | O | 거래처 유사도 계산 |
| 3 | 시도명 | O | 거래처 유사도 계산 |
| 4 | 시군구명 | O | 거래처 유사도 계산 |
| 5 | 도로명주소 | O | 거래처 유사도 계산 |
| 6 | 전화번호 | O | 거래처 유사도 계산 |
| 7 | 고객코드 | X | 고객 고유 키 |
| 8 | 고객명 | O | 100% 일치 여부 판정 |
| 9 | 진료과목명 | X | 동명이인 구분 참고 (유사도 미사용) |

> `거래처코드`, `고객코드`, `진료과목명`은 매칭 점수 계산에 사용되지 않으며, 결과 파일(C.xlsx)에 A/B 원본값으로 포함됩니다.

---

## 7. 실행 방법

### 방법 1: 더블클릭 실행 (권장)

1. `input` 폴더에 `A.xlsx`, `B.xlsx`를 준비합니다.
2. `excel_matcher.exe`를 **더블클릭**합니다.
3. 검은색 콘솔 창에서 진행 상황을 확인합니다.
4. 완료 메시지 확인 후 아무 키나 눌러 종료합니다.

### 방법 2: 명령프롬프트에서 실행

1. `Win + R` → `cmd` 입력 후 Enter
2. 프로그램 폴더로 이동:

```bat
cd C:\경로\excel_matcher
```

3. 실행:

```bat
excel_matcher.exe
```

또는 `run.bat` 더블클릭/실행:

```bat
run.bat
```

## 8. 실행 결과

### 성공 시 생성 파일

| 경로 | 설명 |
|------|------|
| `output/C.xlsx` | 매칭 결과 파일 |
| `logs/run_YYYYMMDD_HHMMSS.log` | 실행 로그 |

### C.xlsx 주요 컬럼

- `(A)...` : A 파일 원본 값 (거래처코드, 거래처명, 시도명, 시군구명, 도로명주소, 전화번호, 고객코드, 고객명, 진료과목명)
- `(B)...` : 매칭된 B 파일 값 (동일 9개 컬럼)
- `거래처 기본 유사도점수`
- `고객명 가점`
- `거래처 최종 유사도점수`
- `거래처 일치여부` : 매칭 / 검수필요 / 미매칭
- `고객명 일치여부` : 일치 / 불일치
- `근접후보1~3...` : 대표 후보와 점수 차이 5점 이내 후보

---

## 9. 결과 해석 가이드

### 거래처 일치여부

| 값 | 의미 |
|----|------|
| 매칭 | 90점 이상, 자동 매칭 권장 |
| 검수필요 | 80~89점, 담당자 확인 권장 |
| 미매칭 | 80점 미만, 다른 거래처일 가능성 높음 |

### 고객명 일치여부

| 값 | 의미 |
|----|------|
| 일치 | 정규화 후 고객명 완전 일치 |
| 불일치 | 고객명 다름 (별도 확인 필요) |

> 거래처가 매칭되어도 고객명은 불일치일 수 있습니다. `진료과목명`은 동명이인 구분 참고용이며, 일치 판정에는 사용되지 않습니다. 두 컬럼을 함께 확인하세요.

## 10. 자주 발생하는 오류와 해결 방법

### 10-1. A.xlsx 없음

**메시지 예시**

```text
입력 파일 A.xlsx 를 찾을 수 없습니다.
```

**해결**

- `input` 폴더에 `A.xlsx` 파일이 있는지 확인
- 파일명 철자 확인 (`a.xlsx`, `A.xls` 등은 사용 불가)

---

### 10-2. B.xlsx 없음

**메시지 예시**

```text
입력 파일 B.xlsx 를 찾을 수 없습니다.
```

**해결**

- `input` 폴더에 `B.xlsx` 파일이 있는지 확인
- 파일명이 정확히 `B.xlsx`인지 확인

---

### 10-3. 필수 컬럼 누락

**메시지 예시**

```text
A.xlsx 파일에 필수 컬럼이 누락되었습니다: 거래처명, 전화번호
```

**해결**

- A, B 파일 모두 9개 필수 컬럼명이 정확한지 확인
- 컬럼명 앞뒤 공백, 오타 여부 확인

---

### 10-4. 엑셀 파일이 열려 있음 (저장 실패)

**메시지 예시**

```text
결과 파일 저장에 실패했습니다. output/C.xlsx 파일이 다른 프로그램(엑셀 등)에서 열려 있는지 확인...
```

**해결**

- 엑셀에서 `output/C.xlsx` 파일을 닫기
- 프로그램 다시 실행

---

### 10-5. rules JSON 파일 없음

**메시지 예시**

```text
규칙 파일이 없습니다: ...\rules\sido_alias.json
```

**해결**

- `excel_matcher.exe`와 같은 폴더에 `rules` 폴더가 있는지 확인
- `sido_alias.json`, `company_remove_words.json` 파일 존재 확인
- 배포 ZIP을 다시 압축 해제하여 누락 파일 복구

---

### 10-6. JSON 파싱 오류

**메시지 예시**

```text
규칙 파일 JSON 형식이 올바르지 않습니다
```

**해결**

- JSON 파일의 `{`, `}`, `[`, `]`, `"`, `,` 문법 확인
- 메모장 대신 VS Code 등 JSON 검증 가능한 편집기 사용
- 원본 JSON 파일로 교체

---

## 11. 운영 팁

- 대용량 파일은 실행 시간이 길어질 수 있습니다. 콘솔의 **진행률**을 확인하세요.
- `검수필요` 건은 `근접후보1~3` 컬럼과 함께 검토하면 빠릅니다.
- 시도명/거래처명 규칙 변경은 `rules` 폴더 JSON만 수정하면 됩니다 (exe 재빌드 불필요).
